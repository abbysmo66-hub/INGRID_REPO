### 📋 1. Objetivos del Trabajo
* Implementar algoritmos utilizando la estructura secuencial pura (**Entrada-Proceso-Salida**).
* Demostrar el uso correcto de operadores aritméticos complejos (`MOD`, `^`) y funciones de truncamiento (`TRUNC`).
* Aplicar estándares profesionales de codificación y buenas prácticas de legibilidad.

### 📝 2. Consignas

### Ejercicio 1: Gestión de Stock y Rentabilidad
Una ferretería desea automatizar el cálculo de sus ventas. El programa debe solicitar el **precio de costo** de un producto y la **cantidad de unidades** vendidas. Se debe calcular y mostrar:
* El precio de venta al público (Costo + 40% de margen de ganancia).
* El valor del IVA (21%) aplicado sobre el precio de venta.
* El monto total final recaudado por la venta de todas las unidades (incluyendo IVA).

### Ejercicio 2: Descomposición de Tiempo (Cronómetro)
Escribir un programa que permita ingresar una cantidad entera de **segundos**. El algoritmo debe convertir y mostrar ese tiempo desglosado exactamente en: **Horas, Minutos y Segundos**.  
*(Ejemplo de entrada: 3700 segundos. Salida esperada: 1 Hora, 1 Minuto, 40 Segundos).*

### Ejercicio 3: Geometría Analítica Aplicada
Un arquitecto necesita calcular los materiales para un terreno circular. Dado el **radio** del terreno (en metros), el programa debe informar:
* La superficie total del terreno (Área).
* La longitud del cerco perimetral (Perímetro).
* La longitud de la diagonal del cuadrado que quedaría perfectamente inscrito dentro de ese círculo.

### Ejercicio 4: Resolución de Ecuación Cuadrática
Desarrollar un algoritmo que permita hallar las dos raíces reales ($x_1$ y $x_2$) de un polinomio de segundo grado de la forma $ax^2 + bx + c = 0$.
* **Entrada:** El usuario debe ingresar los coeficientes **a**, **b** y **c**.
* **Proceso:** Aplicar la fórmula general (Bhaskara).
* **Salida:** Mostrar ambos resultados por pantalla.
> **Nota:** Para este TP no es necesario validar si el discriminante es negativo; asuma que el usuario ingresará valores que den como resultado raíces reales.

### Ejercicio 5: Optimización de Cajero Automático
Simular un sistema de pago de sueldos. Dado un monto entero de dinero (Sueldo Neto), el programa debe indicar la **menor cantidad de billetes** necesarios para cubrir esa suma, utilizando las siguientes denominaciones:
* Billetes de $2000, $1000, $500 y $100.  
> **Restricción:** No usar estructuras condicionales (`Si-Entonces`) ni ciclos. El monto ingresado siempre será múltiplo de 100.

### 🌟 3. DESAFÍO EXTRA (Nivel Universitario)

### Ejercicio 6: Criptografía de Posición
Diseñar un algoritmo para generar un código de seguridad basado en un número entero de **4 dígitos** ingresado por el usuario (ejemplo: `4318`). El programa debe:
1.  **Invertir el orden** de las cifras (ejemplo: de `4318` a `8134`).
2.  **Incrementar en 1** cada dígito individualmente (ejemplo final: `9245`).

*Requisito: Lograr la transformación utilizando exclusivamente operaciones matemáticas (`MOD`, `TRUNC`, `*`, `+`). No se permite el uso de funciones de texto ni condicionales.*
